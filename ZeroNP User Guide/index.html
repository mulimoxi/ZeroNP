aZeroNP User GuideZeroNP Team: Jiyuan Tan, Tianhao Liu, Jinsong LiuSection 1 Brief Introduction ZeroNP is a derivative-free solver proposed by Professor Yinyu Ye for general nonlinear constrained problems in 1989 (Origin name ZERONP), which was first implemented in Matlab. It solves the following problem,
minx f(x)
s.t. g(x)=0,
lh⩽h(x)⩽uh,
lx⩽x⩽ux,
minxf(x)s.t.g(x)=0,lh⩽h(x)⩽uh,lx⩽x⩽ux,where f:Rn→R,g:Rn→Rm1,h:Rn→Rm2f:Rn→R,g:Rn→Rm1,h:Rn→Rm2 are smooth functions. The solver is designed for black-box optimization, where the user cannot provide an explicit analytic expression. We suggest that users should not use ZeroNP for problems with explicit formulas as other (first-order or second-order) methods may be more efficient than the zero-order methods. ZeroNP is suitable for the following situations. • The user only has access to the value of the functions.• There are some noises when calculating the functions. Based on the original code of Professor Ye, ZeroNP is implemented by the ZeroNP team in ANSI C with some improvements. Section 3 ZeroNP Usage (a) Function File The user should provide a function that returns the objective and constraint function value at given point pp. The input and output of the function is as following.In Matlab and Python interface,
function f = cost(p)
pp is the input variable used to evaluate the function values. p(:,i)p(:,i) is the ii-th points. The output ff is a vector. f(1)f(1) is the value of the objective function evaluated at xx. f(2)f(2) through f(m1+1)f(m1+1) are the values of the equality constraints, and f(m1+2)f(m1+2) through f(m1+m2+1)f(m1+m2+1) are the values of the inequality constraints. The order of the inequality constraints should be consistent with the input. For gradient/Hessian, the output of the function should be a column vector gg which contains the gradient/Hessian of objective, equality constraints and inequality constraints in order. The Hessian matrix should be stored in a column-first way.
function f = grad(p) or function f = Hessian(p)
 These functions can be stored in a structure as input(See (b) for details).(b) Input The user can use the following command to call ZeroNP.C
ZERONP_PLUS( ZERONPProb *prob, ZERONPSettings *options, ZERONPSol *output, ZERONPInfo *info, cost_temple *cost_fun, g_temple *grad_fun, h_temple *hess_fun, zeronp_float *l, zeronp_float *h)
grad_fun and hess_fun are the the gradient and Hessian of objective and constraint functions. If the users would like to use zero-order method, they can set the variables to be ZERONP_NULL. l and h are initiate values of the Lagrangian multiplier and Hessian, which can also be set to ZERONP_NULL if the users do not have any information. prob and options are introduced below. output and info are the output information of ZeroNP, which are introduced in output section. The user need to free prob，options, output and info at the end of the program. See the example test file for details.Matlab:
info = ZeroNP(prob, options,fun);
Python:
from pyzeronp import ZERONPmyzeronp = ZERONP(prob=prob, cost=cost, option = options)info = myzeronp.run()
where prob and options are structures/dictionary that contains the information of the problem and parameters of the algorithm. fun structure/dictionary contains the information of objective function. The fields of prob are • prob.pbl is the lower bound of the box constraints (lxlx in (3)). The default setting is -∞-∞.• prob.pbu is the upper bound of the box constraints (uxux in (3)).The default setting is +∞+∞.• prob.ibl is the lower bound of the inequality constraints (lhlh in (3)). The default setting is -∞-∞.• prob.ibu is the upper bound of the inequality constraints (uhuh in (3)).The default setting is +∞+∞.• prob.p0 is the initiate point of the algorithm. It should be within the interior of the box constraints. If the user does not provide initiate point and both prob.pbl and prob.pbu are finite, the default initiate point is set to be the center point of the box, i.e., (prob.pbl + prob.pbu)/2.• prob.ib0 is the estimated inequality value of the inequality constraints. It should be within the interior of the inequality constraints. If the user does not provide the estimation and both prob.ibl and prob.ibu are finite, the default estimation is set to be the center point of the box, i.e., (prob.ibl + prob.ibu)/2.For example, if we want to solve a equality constrained problem with initiate point (1,1,2,3)(1,1,2,3), we may set
prob.p0 = [1,1,2,3]';
If we want to solve a problem with inequality constraints and box constraints, we may set
prob.pbl = l_x; prob.pbu = u_x;prob.ibl = l_h;prob.ibu = u_h;

If you are not familiar with the code, we suggest that you only change the basic setting. Basic Settings:• options.tol: Convergence tolerance. The algorithm will stop when the relative difference is less than options.tol. Default value 10-410-4.• options.tol_con: Tolerance of infeasibility. Default value 5050.• options.min_iter: Maximum number of inner iterations. Default value 1010.• options.cen_diff: If equal to 1, the algorithm uses center difference to calculate gradient. Default value 00. We suggest using center difference with block coordinate descent (See options.rs).• options.noise: whether there are noises when evaluating function value. options.noise = 11 means there are noises and the algorithm will choose to use adaptive delta when computing approximated gradient. options.noise = 2 means calculating a proper step size by estimating the noise level. options.noise = 3 combines previous two methods, using the two methods alternately. options.noise = 00 means there are no noise and the algorithm will use fixed delta. Default value 1.1.• options.delta_end: The lower bound of delta. If the user chooses to use adaptive delta, delta will not be lower than options.delta_end during calculation. Default value 10-510-5.• options.rs: Use random sampling in gradient estimation or not. options.rs=1 means using random sampling. options.rs=2 means using block coordinate. Default value options.rs = 00. Random sampling can reduce function cost. However, turn on this option in constrained optimization problems may lead to poor performance. We suggest to turn on this option in large-scale unconstrained optimization problem. Use options.gd_step to change the step size. The default step size is 0.10.1.• options.batchsize: If you choose to use random sampling in gradient estimation, the algorithm will use options.batchsize samples to estimate gradient each time. Default value options.batchsize = max(1,min(dim/4,50))max(1,min(dim/4,50)).• options.rescue: If you don't have an objective function or you just want to find a feasible solution of the problem, set options.rescue = 1. ZeroNP uses gradient of augmented Lagrange function to decide whether to restart. There is no guarantee that the final solution is the global solution. Restart ZeroNP if it terminates at the maximum number of major iterations.
